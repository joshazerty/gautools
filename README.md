# gaussian-scripts

Python tools for managing [Gaussian 16](https://gaussian.com/) quantum chemistry calculations. Scripts cover the full transition-state search workflow: geometry conversion, relaxed scan analysis, TS optimisation, IRC path following, and endpoint optimisation.

---

## Typical workflow

```
XYZ files  ──►  xyz2inp.py  ──►  Gaussian Opt/Freq
                                        │
                              scan2qst2.py  (relaxed scan → QST2 guess)
                                        │
                                  Gaussian QST2/QST3/TS
                                        │
                                   ts2irc.py  ──►  Gaussian IRC
                                                         │
                                                   irc2opt.py  ──►  Gaussian Opt+Freq
                                                                     (fwd & rev endpoints)
```

At any stage, `gau2xyz.py` can pull a geometry out of any log file.

---

## Scripts

### `gau2xyz.py`
Extracts the final geometry from one or more Gaussian log files and writes XYZ files.

```bash
python gau2xyz.py calc.log
python gau2xyz.py ts1.log ts2.log ts3.log   # batch
```

- Works regardless of termination status — always extracts the last geometry available.
- Prefers Standard orientation; falls back to Input orientation.
- Output: `<basename>.xyz`

---

### `ts2irc.py`
Takes a converged TS (from an Opt, QST2, or QST3 calculation) and sets up an IRC input file.

```bash
python ts2irc.py ts_opt.log
```

- Reads the route section, basis set, charge/multiplicity, and footer from the matching `.inp`/`.gjf`/`.com` file.
- Removes `Opt`, `Freq`, and `QST` keywords; appends `IRC=(CalcFC,MaxPoints=30,StepSize=10)`.
- Strips common suffixes from the output name (`_opt`, `_ts`, `_qst2`, `_freq`, etc.).
- Output: `<basename>_irc.inp`

---

### `irc2opt.py`
Takes a finished IRC log and creates optimisation input files for both the forward and reverse endpoints.

```bash
python irc2opt.py ts_irc.log
```

- Reads route/basis/charge/footer from the matching `.inp` (the IRC input generated by `ts2irc.py`).
- Detects the forward and reverse IRC directions using Gaussian's `Following the Reaction Path in the FORWARD/REVERSE direction` markers for reliable endpoint assignment.
- Route is modified to replace `IRC` with `Opt=(CalcFC,MaxCycles=200) Freq`.
- Output: `<basename>_irc_fwd.inp` and `<basename>_irc_rev.inp`

---

### `scan2qst2.py`
Parses a Gaussian relaxed scan log, plots the potential energy surface, and optionally generates a QST2 input from the scan maximum.

```bash
python scan2qst2.py scan.log
```

- Uses [cclib](https://cclib.github.io/) for robust log parsing.
- Supports Bond, Angle, and Dihedral scans (coordinate type and atoms are read from the matching `.inp` file).
- Plots energy in kcal mol⁻¹ vs. the scanned coordinate (Å or degrees).
- On closing the plot, prompts whether to write a QST2 input using the geometries either side of the maximum.
- Output: `<basename>_qst2.gjf`

---

### `xyz2inp.py`
Converts one or more XYZ files into Gaussian input files ready for submission.

```bash
python xyz2inp.py mol.xyz
python xyz2inp.py conf1.xyz conf2.xyz conf3.xyz   # batch
```

- Output: `<basename>.inp` for each input file.

> **Note:** The route section, charge/multiplicity, and basis set footer in `xyz2inp.py` are set for the Re/C/H/O system with B3LYP-D3BJ/def2SVP+SDD in PCM(1-Pentanol). Edit the `header` and `footer` strings in the script for your own system. The `subgau16` submission script will overwrite `%mem`, `%NProcShared`, and `%chk` automatically.

## Dependencies

| Script | Extra dependencies |
|---|---|
| `gau2xyz.py` | none (stdlib only) |
| `ts2irc.py`  | none (stdlib only) |
| `irc2opt.py` | none (stdlib only) |
| `xyz2inp.py` | none (stdlib only) |
| `scan2qst2.py` | `cclib`, `numpy`, `matplotlib` |

Install optional dependencies:
```bash
pip install cclib numpy matplotlib
# or
conda install -c conda-forge cclib numpy matplotlib
```

---

## Notes

- All scripts expect the Gaussian **input file** (`.inp`, `.gjf`, or `.com`) to share the same basename as the log file and be in the same directory. This is how route sections, basis set footers, and charge/multiplicity are inherited through the workflow without hardcoding.
- `%mem`, `%NProcShared`, and `%chk` in generated input files are placeholders — the `subgau16` submission script replaces them at job submission time.
